  • Giggle Search

    As an interface for the algorihtm Giggle this search engine takes user inputted chromosone regions and finds overlapping annotation files in genomic databases, such as the UCSC Genomic Institute.

  • HOW TO: Text Input Interval Search

    Enter interval/s of interest in designated format:
    - General Format for Single Interval: chr(#): (lower bound base position) - (upper bound base position) from (source)
    - General Format for Multi Interval: chr(#): (lower bound base position) - (upper bound base position), (lower bound base position) - (upper bound base position), ... ,chr(#):(lower bound base position) - (upper bound base position) from (source)


    The parser will take variable formats, for example the ":" or "-" seperating the chr and the bounds just needs to be some symbol.

    But to be valid:
    1. The chr(#) must not have a space between the number and the string "chr"
    2. The chromosome/bounds must each have a seperator between them (either a space or symbol)
    3. The last word in the query must be the source name code as specified below.
    4. Extra spaces or symbols will be removed during parsing

    chr2:1000-2000 from UCSC will be parsed the same as chr2??1000---2000 FrOmm UCSC. But just because you CAN, does not mean you SHOULD.

    It will be easiest and most accurate if you follow the pattern format below. Here are some examples:

    Example Input Interpretation
    Single Interval ch1: 100-2000 from UCSC Finds overlap in bases 100 to 2000 in Chromosome 1 with files from UCSC.
    Multi Interval & Single Chromosome chr1:100-2000, 150000-160000 from UCSC
    or equivalently
    chr1:100-2000, chr1:50000-160000 from UCSC    		 Finds overlap in bases 100 to 2000 in Chromosome 1 or bases 150000 to 160000in Chromosome 1 with files from UCSC.
    #1 Multi Interval & Chromosome chr2:100-2000, chr1:50000-160000 from UCSC Finds overlap in bases 100 to 2000 in Chromosome 1, or bases 100 to 2000 in Chromosome 1 with files from UCSC.
    #2 Multi Interval & Chromosome chr2:100-2000,5000-60000, chr1:50000-160000 in UCSC Finds overlap in bases 50000 to 160000 in Chromosome 1, or bases 100 to 2000 in Chromosome 2, or bases 5000 to 60000 in Chromosome 2 with files from UCSC.

    ** Important Note about Multi Interval search: entering multiple intervals does not return combined overlapping bases, instead it finds overlap for each interval and allows you to toggle between the intervals entered

    Current Restrictions (Subject to Change with New Versions)
    Available Data Sources UCSC: UCSC Human Feb. 2009 (GRCh37/hg19) Assembly
    Max Interval Width One megabase or 1000000 bases
    Chromosome Bounds Base positions should not exceed chromosome size bounds, they should also not be negative.

  • Additional Features

    UCSC Genome Browser Connection If UCSC is the inputted source a toggle will appear below the search bar to allow for top # of overlapping files to be displayed in the UCSC Genome Track browser
